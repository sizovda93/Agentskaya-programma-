→

4 типа уведомлений

  • Смена статуса лида
  • Новая выплата (лид → won)
  • Смена статуса выплаты
  • Сообщение от менеджера

Обработка ошибок

  • Бот заблокирован (403) → привязка деактивируется
  • Ошибки Telegram не блокируют веб-ответ
  • Всё логируется в audit_logs

Рефералы

Реферальная система

Как работает

Агент получает 8-символьный код
→
Делится ссылкой /?ref=CODE
→
Клик отслеживается
→
Менеджер создаёт лид с ref_code

Дедупликация

  • SHA256(IP + refCode) → ip_hash
  • Окно уникальности: 1 час
  • Конверсия = leads / unique clicks × 100%

Представления

Агент

Свой код, ссылка, статистика (клики, уникальные, конверсия), шаблоны для мессенджеров и соцсетей

Менеджер / Админ

Лидерборд: все агенты, отсортированные по количеству реферальных лидов и конверсии

Безопасность

Защита данных

🔒

Аутентификация

JWT в httpOnly cookie (7 дней TTL). Пароли: bcrypt, 12 раундов. Middleware проверяет каждый запрос.

🛡

Авторизация

Role-based access control (RBAC). Middleware + requireRole() на каждом API. Каждая роль видит только своё.

📚

Данные

Параметризованные SQL-запросы. Whitelist расширений при загрузке (макс 10 МБ). Path traversal protection.

Аудит-лог

Все действия записываются: login, register, lead.created, lead.updated, payout.status_changed, telegram.linked, settings.updated и другие.

Финансы

State Machine выплат

pending
Ожидает
→
processing
В обработке
→
paid
Оплачено (Admin)
pending / processing
→
rejected
+ причина

Правила

  • Только Admin может поставить статус «Оплачено»
  • При отклонении обязательно указывается причина (rejection_reason)
  • paid и rejected — финальные статусы, переход из них невозможен
  • Все смены статуса логируются + Telegram-уведомление агенту
